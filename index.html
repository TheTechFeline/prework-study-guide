HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • The title element specifies a title for the webpage and is shown in the browser's title bar or page's tab
  • The meta element tells the browser more information about a page and is not viewable to a site visitor
  • The header element typically contains a group of introductory or navigational aids
  • The footer element represents a footer for its nearst sectioning content or sectioning root element; people often put copyright notices here
  • The section alement represents a standable section of functionality contained within an HTML document

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A border surrounds the padding and content within an element.
  • The box-sizing property allows us to include the padding and border in an element's total width and height.

Git

  • git status: checks what branch we are currently in
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add . or git add -A: stages all changes in the current directory for the next commit
  • git commit -m "commit message": commits the staged snapshot with a brief log message
  • git push origin branch-name: pushes all committed changes in the local branch to the remote repository
  • git pull origin branch-name: fetches changes from the remote repository and merges it with the current branch
  • git branch: lists all local branches in the current repository
  • git merge branch-name: merges the specified branch's history into the current branch
